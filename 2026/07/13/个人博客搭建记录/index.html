发表于2026-07-13|更新于2026-07-14
|浏览量:

搭建篇

基于Hexo框架搭建github博客，主机是win11

官方指南 and 参考视频

首先安装 git 和 Node.js，然后去学习一下git 和 github 之间的交互

首先新建文件夹Local repository，右键点击 Git Bash Here

拉取项目就点击github项目主页绿绿的Code，找到SSH即可

新建文件夹 MyBlog，进入Bash，执行npm install -g hexo-cli安装Hexo工具

接下来初始化博客

hexo server 启动服务，http://localhost:4000/本地游览器访问，出现界面就成功了

然后创建一个github仓库，名字必须是github用户名.github.io，勾选Add a README file

创建后点击settings，在pages中就能看到自己的个人网站了！

回到\MyBlog\myblogs 找到_config.yml，用VS打开，定位到最后

repo格式通过上文提到的拉项目方法获取

关于branch的填写，必须要和博客 settings 中GitHub Pages部分指定的Branch保持一致。

在\MyBlog\mybgs，使用 git bash输入

出现INFO Deploy done: git后说明推送成功，这样就搭建好了

\MyBlog\myblogs\source\_posts 下写md，然后执行下面代码更新(需要时间)

这样我们就安装好了butterfly主题

然后按自己的需求进行配置，参考配置修改，由于细节太多和可复用性弱，请自行查阅喵~

删除myblog 下面的_config.landscape.yml无影响
把butterfly的配置文件（放在开头）复制到博客根目录，并重命名为 _config.butterfly.yml，这样以后升级主题时就不会覆盖你的自定义设置

主题界面

标签页

找到 source/tags/index.md 這個文件

参数 解释
type 【必須】頁面類型，必須為 tags
orderby 【可選】排序方式 ：
random - 隨機排序 / name - 標簽名字排序 / length - 標簽數量排序
order 【可選】排序次序： 1（升序），-1（降序）

hexo server 本地查看http://localhost:4000/tags/

分类页

找到 source/categories/index.md 這個文件

本地查看http://localhost:4000/categories/

友链

找到 source/link/index.md 這個文件

添加友链在 source/_data/link.yml中

404

_config.butterfly.yml 找到error_404 设置成true即可
